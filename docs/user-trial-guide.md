# 朋友制作器试用说明

专为 Switch 版《朋友收集：梦想生活》制作。

这份文档是给第一次上手的试用者准备的。目标只有一个：

`把系统启动起来 -> 连接 ESP32 -> 连接 Switch -> 选择图片 -> 开始绘制`

## 开始前必须确认

开始绘制前，请一定先确认下面 3 件事：

1. `画笔大小` 已经在 Switch 绘图页里手动切到和网页一致  
例如网页选的是 `3 号笔`，Switch 里也必须先手动切到 `3 号笔`。

2. `光标/画笔` 已经停在画布中心  
当前系统默认按“从画布中心起步”生成命令，如果起点不在中心，整张图会偏移。

3. 如果使用 `官方色绘制`  
开始前请先手动把右侧 `9` 个色盘槽位，全部设成 **基本颜色页左上角的白色**。  
这一步非常重要，因为当前官方色模式默认把这 9 个槽位都当成“白色起点”来配置。

## 1. 试用前准备

你需要准备：

- 一台 `macOS` 电脑
- 一块 `ESP32-WROOM-32 / ESP-32S` 开发板
- 一台 `Nintendo Switch`
- 一根能正常传数据的 USB 线
- 已安装 `Node.js`
- 已安装 `PlatformIO`

建议版本：

- `Node.js 20+`
- `npm 10+`
- `PlatformIO Core 6+`

## 2. 首次安装

打开终端，进入项目目录：

```bash
cd /Users/xiyu/Documents/code/friendmaker
```

安装依赖：

```bash
npm install
```

检查 TypeScript 是否正常：

```bash
npm run check
```

如果这里没有报错，就说明桌面端已经能启动。

## 3. 刷入 ESP32 固件

先把开发板接到 Mac。

进入固件目录：

```bash
cd /Users/xiyu/Documents/code/friendmaker/firmware/esp32
```

刷入推荐环境：

```bash
~/.platformio/penv/bin/pio run -e esp32dev_wireless -t upload
```

如果你知道自己的串口，也可以明确指定：

```bash
~/.platformio/penv/bin/pio run -e esp32dev_wireless -t upload --upload-port /dev/cu.usbserial-0001
```

刷入成功后，可以用下面这条看启动日志：

```bash
~/.platformio/penv/bin/pio device monitor -b 115200
```

看完后按 `Ctrl+C` 退出串口监视器。  
注意：**开着串口监视器时，网页无法同时连接这个串口。**

## 4. 启动网页系统

回到项目根目录：

看到这行说明成功：

## 5. 第一次使用顺序

网页一共 3 页：

- `脚本生成`
- `刷入固件`
- `手柄测试`

第一次使用建议按这个顺序：

1. `刷入固件`
2. `手柄测试`
3. `脚本生成`

### 5.1 刷入固件页

在 `刷入固件` 页：

- 选择环境：`ESP32-WROOM-32 / ESP-32S`
- 选择串口设备
- 点击 `编译并刷入固件`

右侧状态卡显示成功后再进入下一步。

### 5.2 手柄测试页

在 `手柄测试` 页：

- 先点击 `连接手柄`
- 去 Switch 上打开：
  - `控制器`
  - `更改握法/顺序`

如果连接正常，页面状态会逐渐进入“已连接 / 可发送”。

如果连接中途断掉：

- 先点 `重置蓝牙`
- 等待完成
- 再点 `连接手柄`

### 5.3 脚本生成页

确认手柄状态正常后，再去 `脚本生成` 页。

当前可选模式：

- `单色绘制`
- `官方色绘制`

基础操作：

1. 导入图片
2. 选择画笔大小
3. 选择绘制模式
4. 需要时调整色阶数量
5. 点击 `仅生成命令` 先看预览
6. 确认无误后点击 `一键开始绘制`

开始绘制前再检查一次：

- Switch 里的画笔大小，和网页当前选择一致
- 画笔停在画布中心
- 如果是 `官方色绘制`，9 个色盘槽位都已经手动设成 **左上角白色**

## 6. 当前推荐试用方式

为了提高成功率，第一次试用建议这样选：

- 模式：`单色绘制`
- 画笔：`3`
- 透明背景 PNG 优先
- 图案尽量简单
- 先用主体清晰、颜色块大的图片

如果你要试彩色：

- 优先试 `官方色绘制`
- 开始前，先手动把 `9` 个槽位都设成 **基本颜色页左上角的白色**
- 开始前，再确认画笔已经切到正确大小，并且停在画布中心

## 7. 当前已知规则

当前系统按这些固定假设运行：

- 画布按 `250x250` 处理
- 进入绘图页后光标在画布中心
- 开始绘制前，画笔已经手动切到网页选择的正确大小
- `A` 负责绘制
- 方向键负责移动
- `1 / 3 / 7 / 13 / 19 / 27` 六种画笔大小都已接入

## 8. 使用中的注意事项

- 绘制过程中不要拔掉开发板
- 绘制过程中不要关闭 `npm run ui:dev` 那个终端
- 只关闭浏览器标签页，一般不会立刻中断绘制
- 但如果把本地服务进程关掉，后续命令就不会再继续发送

## 9. 常见问题

### 9.1 页面里看不到串口

先重新插拔开发板，然后点击：

```txt
刷新串口
```

如果还不行，可以在终端里先看系统有没有识别到：

```bash
npm run dev -- --list-ports
```

### 9.2 提示串口被占用

通常是你还开着串口监视器。先关闭：

- `pio device monitor`
- Arduino 串口监视器
- 其它串口助手

再回网页重试。

### 9.3 手柄断链后连不上

按这个顺序：

1. 点击 `重置蓝牙`
2. 等待完成
3. 点击 `连接手柄`
4. 在 Switch 上重新进入 `更改握法/顺序`

### 9.4 画出来的颜色和预览不完全一样

这是当前版本的已知情况。  
原因通常不是绘制失败，而是：

- 游戏内官方色盘和网页里的近似色表还没完全校准
- 不同画笔大小下，实机观感会和网页预览略有差异
