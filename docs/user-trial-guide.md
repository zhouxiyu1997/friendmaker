# 朋友制作器试用说明

专为 Switch 版《朋友收集：梦想生活》制作。

这份文档是给第一次上手的试用者准备的。目标只有一个：

`把网页启动起来 -> 刷入固件 -> 连接 Switch -> 选择图片 -> 开始绘制`

## 平台支持

当前试用版支持：

- `macOS`：完整试用流程
- `Windows`：手动安装与手动启动流程

说明：

- `macOS` 仍然是当前最完整、最省事的版本
- `Windows` 现在可以手动安装并试用，但还没有一键启动器
- `Linux` 暂未完成联调

## 路径说明

下面文档里出现的路径都是示例值，不是固定值：

- 把 `/path/to/friendmaker` 替换成你自己的项目目录
- 把 `<your-serial-port>` 替换成你自己的串口设备，例如 macOS 下的 `/dev/cu.usbserial-0001` 或 Windows 下的 `COM3`
- 如果 `pio` 已经在你的 shell `PATH` 里，可以直接写 `pio ...`
- 如果没有，就继续用完整路径：
  - macOS：`~/.platformio/penv/bin/pio ...`
  - Windows：`%USERPROFILE%\\.platformio\\penv\\Scripts\\pio.exe ...`

## 先看整体使用方式

### 基本原理

朋友制作器的自动绘画原理，是通过数据线连接电脑与 `ESP32` 开发板，先把固定固件刷入开发板，再由电脑发送绘图指令，开发板按规律模拟 Switch 手柄输入，从而完成自动绘画。

可以把它理解成一条固定链路：

`电脑生成绘图命令 -> 串口发送给 ESP32 -> ESP32 模拟手柄输入 -> Switch 画布自动绘制`

### 当前已实现功能

当前版本已经支持：

- 导入 `PNG / JPG / SVG` 图片，并生成绘制预览
- 支持 `1 / 3 / 7 / 13 / 19 / 27` 六种画笔大小
- 支持 `单色绘制`
- 支持 `官方色绘制`
- 支持先点击 `仅生成命令` 查看预览与脚本
- 支持点击 `一键开始绘制` 直接执行
- 支持在网页里完成 `刷入固件`、`手柄测试` 和 `脚本生成`

说明：

- 当前彩色模式请使用 `官方色绘制`
- `自定义调色板绘制` 暂未开放

### 推荐使用顺序

建议第一次按下面顺序使用：

1. 用数据线连接电脑和开发板
2. 找到正确串口并先刷入固件
3. 进入 `手柄测试` 页，确认虚拟手柄连接和按键测试正常
4. 手柄测试通过后，再回到 `脚本生成` 页导入图片
5. 先点击 `仅生成命令` 检查预览效果
6. 确认无误后，再点击 `一键开始绘制`

### 重要提醒

- 如果 `手柄测试` 没通过，不要开始正式绘制
- 开始绘制前，Switch 应处于初始进入绘画状态的页面
- 开始绘制前，画笔必须停在画布中心
- 开始绘制前，侧边色彩槽不要手动修改
- 建议优先使用 `方块笔刷`
- 建议第一次先用简单图和大画笔测试，确认链路稳定后再尝试复杂图片
- 开始绘制后，不要再手动操作手柄，也不要触碰屏幕，否则容易导致错位
- 当前虽然有 `暂停绘制` 和 `中断绘制` 功能，但非必要情况下最好不要暂停

## 开始前必须确认

开始绘制前，请一定先确认下面 3 件事：

1. `画笔大小` 已经在 Switch 绘图页里手动切到和网页一致  
例如网页选的是 `3 号笔`，Switch 里也必须先手动切到 `3 号笔`。另外建议把游戏里的笔刷切到 `方块笔刷`，整体观感通常会更美观。

2. `光标/画笔` 已经停在画布中心  
当前系统默认按“从画布中心起步”生成命令，如果起点不在中心，整张图会偏移。

3. 如果使用 `官方色绘制`  
保持游戏默认的 `9` 个色盘槽位初始颜色即可。  
当前官方色模式会按这 9 个默认槽位的原始颜色状态来配置。

## 1. 试用前准备

你需要准备：

- 一台 `macOS` 或 `Windows` 电脑
- 一块 `ESP32-WROOM-32 / ESP-32S` 开发板
- 一台 `Nintendo Switch`
- 一根能正常传数据的 USB 线
- 已安装 `Node.js`
- 已安装 `PlatformIO`

建议版本：

- `Node.js 20+`
- `npm 10+`
- `PlatformIO Core 6+`

## 2. 首次安装

打开终端，进入项目目录：

```bash
cd /path/to/friendmaker
```

安装依赖：

```bash
npm install
```

检查 TypeScript 是否正常：

```bash
npm run check
```

如果这里没有报错，就说明桌面端已经能启动。

## 3. 启动方式

### 3.1 macOS 一键启动

如果你是 `macOS` 用户，现在可以直接双击：

- `Start Friend Maker.command`

或者在终端里运行：

```bash
cd /path/to/friendmaker
./Start\ Friend\ Maker.command
```

这个启动器会自动：

- 检测 `Node.js`、`npm`、`Python 3`、`PlatformIO`
- 尝试自动安装缺失的软件
- 在需要时自动执行 `npm install`
- 启动本地网页系统
- 自动打开 `http://127.0.0.1:4307`

注意：

- 如果机器里没有 `Homebrew`，启动器会先询问是否自动安装
- 第一次安装软件时，终端可能会要求输入系统密码
- 绘制过程中不要关闭这个启动出来的终端窗口
- 启动器现在也会自动补装 `PlatformIO`，这样后续刷固件可以直接使用

### 3.2 Windows 手动启动

如果你是 `Windows` 用户，请先参考：

- [Windows 手动安装说明](setup-windows.md)

最短流程是：

```powershell
cd C:\path\to\friendmaker
npm install
npm run check
npm run ui:dev
```

## 4. 刷入 ESP32 固件

当前推荐优先在网页里完成刷固件，命令行方式作为兜底。

### 4.1 推荐方式：在网页里刷固件

先把开发板接到电脑，然后启动网页系统：

```bash
cd /path/to/friendmaker
npm run ui:dev
```

打开：

```txt
http://127.0.0.1:4307
```

然后进入 `刷入固件` 页：

1. 选择环境：`ESP32-WROOM-32 / ESP-32S`
2. 选择串口设备
3. 点击 `编译并刷入固件`
4. 等待右侧状态卡显示成功

如果页面里提示没有找到 `PlatformIO`，请先确认你已经装好它，或者改用下面的命令行兜底方式。

### 4.2 兜底方式：命令行刷固件

进入固件目录：

```bash
cd /path/to/friendmaker/firmware/esp32
```

刷入推荐环境：

```bash
~/.platformio/penv/bin/pio run -e esp32dev_wireless -t upload
```

如果你知道自己的串口，也可以明确指定：

```bash
~/.platformio/penv/bin/pio run -e esp32dev_wireless -t upload --upload-port <your-serial-port>
```

刷入成功后，可以用下面这条看启动日志：

```bash
~/.platformio/penv/bin/pio device monitor -b 115200
```

看完后按 `Ctrl+C` 退出串口监视器。  
注意：**开着串口监视器时，网页无法同时连接这个串口。**

Windows 示例：

```powershell
cd C:\path\to\friendmaker\firmware\esp32
$env:USERPROFILE\.platformio\penv\Scripts\pio.exe run -e esp32dev_wireless -t upload --upload-port COM3
```

### 5.1 平时如何重新启动项目

如果你只是想把本地网页重新打开，或者上一轮试用结束后想再启动一次，按下面做就可以：

1. 先回到上一次启动项目的终端窗口
2. 按 `Ctrl+C` 停掉当前本地服务
3. 再重新启动项目

`macOS` 最省事的方式：

- 重新双击 `Start Friend Maker.command`

或者继续用命令行方式：

```bash
cd /path/to/friendmaker
npm run ui:dev
```

如果你是 `Windows`，重新启动通常这样做：

```powershell
cd C:\path\to\friendmaker
npm run ui:dev
```

看到下面这行，说明已经重新启动成功：

```txt
Switch Auto Draw UI running at http://127.0.0.1:4307
```

再在浏览器里打开：

```txt
http://127.0.0.1:4307
```

补充说明：

- 如果浏览器页面还开着，通常直接刷新页面就行
- 如果提示端口被占用，通常是旧的终端进程还没关掉，先回原来的终端按一次 `Ctrl+C`
- 如果你是通过 `Start Friend Maker.command` 启动的，也不要同时再开一个 `npm run ui:dev`，避免重复启动
- 如果你是 `Windows`，也不要重复打开多个 `npm run ui:dev` 窗口，否则也可能出现端口占用

### 5.2 GitHub 仓库更新后，如何更新本地版本

如果远程仓库有新版本，你本地建议按这个顺序更新：

1. 先停掉当前项目
2. 进入项目目录
3. 拉取最新代码
4. 重新安装依赖
5. 检查项目
6. 重新启动网页

命令如下：

```bash
cd /path/to/friendmaker
git pull
npm install
npm run check
npm run ui:dev
```

如果你是 `macOS`，也可以在执行完前 4 步以后，直接重新双击：

- `Start Friend Maker.command`

这样会更接近平时试用时的方式。

如果你是 `Windows`，通常这样更新：

```powershell
cd C:\path\to\friendmaker
git pull
npm install
npm run check
npm run ui:dev
```

更新后建议额外注意 3 件事：

1. 如果更新说明里提到 `固件`、`串口协议`、`ACK`、`蓝牙连接`、`ESP32`，最好再去网页的 `刷入固件` 页重新刷一次固件  
不确定要不要重刷时，直接重刷一次通常最稳妥。

2. 如果 `git pull` 提示你有本地改动、无法自动合并，先不要随便删文件  
如果这些改动是你自己改过的内容，先备份，或者先问维护者该怎么处理。

3. 如果 `npm install` 后依赖有更新，第一次重新启动可能会稍慢一点  
这通常是正常现象。

## 6. 第一次使用顺序

网页一共 3 页：

- `脚本生成`
- `刷入固件`
- `手柄测试`

第一次使用建议按这个顺序：

1. `刷入固件`
2. `手柄测试`
3. `脚本生成`

### 6.1 刷入固件页

在 `刷入固件` 页：

- 选择环境：`ESP32-WROOM-32 / ESP-32S`
- 选择串口设备
- 点击 `编译并刷入固件`

右侧状态卡显示成功后再进入下一步。

### 6.2 手柄测试页

在 `手柄测试` 页：

- 先点击 `连接手柄`
- 去 Switch 上打开：
  - `控制器`
  - `更改握法/顺序`

如果连接正常，页面状态会逐渐进入“已连接 / 可发送”。

如果连接中途断掉：

- 先点 `重置蓝牙`
- 等待完成
- 再点 `连接手柄`

### 6.3 脚本生成页

确认手柄状态正常后，再去 `脚本生成` 页。

如果 `手柄测试` 还没通过，暂时不要开始正式绘制。

当前可选模式：

- `单色绘制`
- `官方色绘制`

基础操作：

1. 导入图片
2. 选择画笔大小，并确认游戏里已切到 `方块笔刷`
3. 选择绘制模式
4. 需要时调整色阶数量
5. 点击 `仅生成命令` 先看预览与脚本
6. 确认无误后点击 `一键开始绘制`

开始绘制前再检查一次：

- Switch 里的画笔大小，和网页当前选择一致
- Switch 里的笔刷建议切到 `方块笔刷`，整体观感通常会更美观
- 画笔停在画布中心
- 如果是 `官方色绘制`，保持 `9` 个色盘槽位为游戏默认初始颜色

## 7. 当前推荐试用方式

为了提高成功率，第一次试用建议这样选：

- 模式：`单色绘制`
- 画笔：`3`
- 笔刷：`方块笔刷`
- 透明背景 PNG 优先
- 图案尽量简单
- 先用主体清晰、颜色块大的图片

如果你要试彩色：

- 优先试 `官方色绘制`
- 开始前，保持 `9` 个槽位为游戏默认初始颜色
- 开始前，再确认画笔已经切到正确大小，并且停在画布中心

## 8. 当前已知规则

当前系统按这些固定假设运行：

- 画布按 `256x256` 脚本坐标处理
- 进入绘图页后光标在画布中心
- 开始绘制前，画笔已经手动切到网页选择的正确大小
- `A` 负责绘制
- 方向键负责移动
- `1 / 3 / 7 / 13 / 19 / 27` 六种画笔大小都已接入

## 9. 使用中的注意事项

- 绘制过程中不要拔掉开发板
- 绘制过程中不要关闭 `npm run ui:dev` 或一键启动脚本打开的那个终端
- 绘制过程中不要再操作手柄，也不要触碰 Switch 屏幕，否则容易导致错位
- 当前虽然支持 `暂停绘制` 和 `中断绘制`，但非必要情况下尽量不要暂停
- 只关闭浏览器标签页，一般不会立刻中断绘制
- 但如果把本地服务进程关掉，后续命令就不会再继续发送

## 10. 常见问题

### 10.1 页面里看不到串口

先重新插拔开发板，然后点击：

```txt
刷新串口
```

如果还不行，可以在终端里先看系统有没有识别到：

```bash
npm run dev -- --list-ports
```

### 10.2 提示串口被占用

通常是你还开着串口监视器。先关闭：

- `pio device monitor`
- Arduino 串口监视器
- 其它串口助手

再回网页重试。

### 10.3 手柄断链后连不上

按这个顺序：

1. 点击 `重置蓝牙`
2. 等待完成
3. 点击 `连接手柄`
4. 在 Switch 上重新进入 `更改握法/顺序`

### 10.4 画出来的颜色和预览不完全一样

这是当前版本的已知情况。  
原因通常不是绘制失败，而是：

- 游戏内官方色盘和网页里的近似色表还没完全校准
- 不同画笔大小下，实机观感会和网页预览略有差异
