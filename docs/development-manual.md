## 1. 当前目标

当前项目的目标不是一次性做完整的“任意图片自动彩绘”，而是先把下面这条实际试用链路做稳：

`刷入固件 -> 手柄测试 -> 调试测速 -> 网页导图 -> 串口 ACK 执行 -> ESP32 蓝牙控制器输出 -> Switch 画板稳定复现`

当前优先级：

1. 蓝牙连接和长时间执行稳定
2. 单色绘制与官方色绘制稳定
3. 颜色来源与位移假设继续校准
4. 再考虑更复杂的自动化能力

## 2. 当前稳定能力

截至当前版本，已经验证或基本可用的能力有：

- Web UI 已覆盖 `脚本生成 / 刷入固件 / 手柄测试 / 调试测速` 四页
- 网页里可以直接调用 `PlatformIO` 编译并刷入固件
- 网页里可以直接做手柄连接、蓝牙重置、按钮/方向/摇杆测试
- 网页里可以直接调 `inputDelay / buttonPressDuration` 并跑闭环测速
- `256x256` 脚本坐标画布
- `1 / 3 / 7 / 13 / 19 / 27` 六种画笔大小
- 进入绘图页后，从画布中心开始绘制
- 中断恢复后，重新进入绘图页时也只按“画布中心起步”建模
- `A` 用作落笔/绘制
- 方向键用于单格移动
- Web UI 可导入图片、生成预览、查看统计并执行命令
- 单色绘制链路已跑通
- 官方色绘制链路已接入 `image-q`
- 官方色会量化到 `7 x 12` 的 `84` 色基础盘，并配置游戏右侧 `9` 个槽位
- 自动扣背景、辅助线预览、官方色盘预览都已接入
- 执行日志、刷写日志、测试日志都已在网页中可观察
- `inputDelay` 当前更适合作为稳定性旋钮，`buttonPressDuration` 更适合作为力度旋钮
- 暂停、中断或异常后的恢复任务会落盘；如果应用在暂停态退出，下次启动后该任务也会自动转成可恢复状态
- 由于自定义颜色调色仍不稳定，实验性的通用 `多色绘制` 当前继续隐藏
- ESP32 已支持基础串口协议
- ESP32 已支持网页内正式绘制所需的 ACK 执行链路
- ESP32 已支持 `TAP <按钮> <次数>` 测试命令
- ESP32 已支持 `HOLD <按钮> <毫秒>` 测试命令

## 3. 当前绘图页建模

当前目标绘图页按下面的固定规则建模：

- 进入绘图页后，光标在主画布中心
- 当前没有一个可依赖的“把主画布光标稳定归到左上角”的复位语义
- 因此恢复流程也必须假设用户重新进入绘图页后，仍然从画布中心重新起步
- 当前工具是画笔
- `A` 为绘制
- 方向键每次移动 1 格
- 画布按 `256x256` 脚本坐标处理

这意味着脚本生成页当前主线就是：

- 固定 `256x256`
- 固定按“中心起步”
- 单色与官方色双主线

补充区分：

- 文档里如果提到“归左上角”或“左上角起点”，默认都不是指主画布
- 这类描述只应用在色盘列表、基本颜色页或自定义颜色编辑页的内部导航建模上

## 4. 多色方案分叉

当前多色有两条可能路线：

### 路线 A：基本颜色

这是当前推荐主线。

已知事实：

- 基本颜色是系统自带默认盘
- 基本颜色区域为 `7 行 x 12 列`
- 总共 `84` 个可直接选择的基础颜色

这条路线的优点：

- 颜色集合有限
- 不需要在自定义颜色编辑页里精确调色
- 更适合开环控制
- 更适合做稳定的自动绘制

这条路线后续应该这样做：

1. 记录完整 `7 x 12` 基本颜色表
2. 给每个颜色定义固定的 `row / col`
3. 网页端把目标图片量化到这 `84` 个颜色
4. ESP32 只负责打开颜色页并选择对应格子

当前结论：

`基本颜色` 是后续多色绘制的推荐主线。

当前已接入的程序设计：

- Web UI 新增 `官方色绘制`
- 官方色绘制会把图片量化到 `84` 个基本颜色
- 脚本层新增 `BC <slot> <row> <col>` 命令
- 固件会按 `slot -> 基本颜色页 row/col` 的方式配置 9 个色盘槽位
- 当前策略不再尝试“归左上角”，而是假设 9 个槽位起始都处于游戏默认颜色状态，并跟踪每个槽位当前的 `row/col`，按普通四方向移动到目标格子
- 脚本开始时会先发送 `BC RESET`，把固件内部槽位状态重置为“默认槽位颜色状态”
- `BC RESET` 只重置颜色槽位的内部跟踪状态，不表示主画布光标被重置到左上角
- 在基本颜色页移动到目标格子后，按一次 `A` 就会直接回到画布，因此官方色槽位配置不再额外发送 `B / A / B`

### 路线 B：自定义颜色

这是当前已探索但暂缓的路线。

已知事实：

- 颜色页有“基本颜色”和“自定义颜色”两个标签
- 进入绘图页后按 `Y` 打开色盘列表
- 色盘列表有 `9` 个色盘位
- 色盘列表上方还有其他非色盘项
- 再按一次 `Y` 可进入色盘编辑页
- 进入编辑页后，`R` 切到“自定义颜色”标签，`L` 切到“基本颜色”标签
- `B` 退出色盘页/编辑页
- `A` 选中当前色盘
- 编辑页中：
  - `ZL / ZR` 控制下面的色相条
  - 上方大色块是二维选色区
  - `M 1 0 / M -1 0 / M 0 -1 / M 0 1` 的方向语义已确认正确

当前不继续推进自定义颜色自动化的原因：

- `TAP DLEFT 20` 与 `TAP DLEFT 40` 的位移关系不稳定
- 点按次数与光标位移不是严格线性
- 长按和点按的结果不同，长按存在递增或加速效应
- 没有视觉反馈时，无法保证 `#RRGGBB` 被准确落到目标位置

当前结论：

`自定义颜色自动调色` 先暂缓，不作为当前多色主线。

## 5. 自定义颜色页已确认信息

虽然当前暂缓自动改色，但已确认的事实需要保留，后续可能会继续使用。

### 5.1 色盘列表

- `Y` 打开色盘列表
- 色盘列表中，连续按 `下` 可以稳定到底部
- 底部到顶部之间共有 `9` 个色盘位

这说明色盘列表的稳定选择策略应该是：

1. 打开列表
2. 一路向下归到底部
3. 再向上回到目标槽位

### 5.2 编辑页方向语义

已确认：

- 上方大色块：
  - `M 1 0` 向右
  - `M -1 0` 向左
  - `M 0 -1` 向上
  - `M 0 1` 向下
- 下面色相条：
  - `ZL` 向左
  - `ZR` 向右

### 5.3 编辑页重置策略

已确认：

- 自定义颜色流程进入编辑页后，需要先按一次 `R` 才是正确的自定义颜色标签
- 官方色流程进入编辑页后，需要先按一次 `L` 才是正确的基本颜色标签
- 同时按对角方向不稳定
- 先纵向归零，再横向归零，再重置色相条更合理
- 当前自定义颜色实验分支改为优先测试“左上角起点”
- 目前实测稳定的一组归位参数为：上推 `1500ms`、左推 `3000ms`、`ZL` 长按 `2500ms`

推荐的重置顺序是：

1. 向上归零
2. 向左归零
3. `ZL` 长按归到色相条左端

### 5.4 量程探索结论

当前探索得到的现象：

- 上方大色块横向量程明显很长
- 上方大色块纵向量程约为横向的一半量级
- 色相条使用 `HOLD` 测得的时间不可靠，因为长按可能加速
- 目前按实机测试记录，自定义颜色编辑页可先按以下量程理解：
  - 色相约 `200` 步
  - 饱和度约 `213` 步
  - 明度约 `112` 步
- `ZR` 的色相前进方向与标准 `HSV hue` 增长方向相反，换算时需要做反向映射
- 当前归位到左上角后，明度高端更接近起点；要压低明度，需要向下移动而不是向上移动

当前结论：

以后如果要继续做自定义颜色自动化，必须：

- 优先使用 `TAP` 计数
- 不再使用 `HOLD` 时长直接换算颜色坐标

## 6. 当前协议补充

当前协议除原有绘图命令外，还增加了测试型命令：

### 6.1 `TAP`

格式：

```txt
TAP <BUTTON> <COUNT>
```

示例：

```txt
TAP DRIGHT 216
TAP DLEFT 110
TAP ZR 20
```

含义：

- 连续点按某个按钮若干次
- 这是“离散步进”命令
- 适合做编辑页或色相条的步数标定

### 6.2 `HOLD`

格式：

```txt
HOLD <BUTTON> <MS>
```

示例：

```txt
HOLD ZL 10000
HOLD ZR 2000
```

含义：

- 持续按住某个按钮若干毫秒
- 只适合做粗量程观察
- 不适合作为最终精确调色依据

### 6.3 `STICK`

格式：

```txt
STICK <X> <Y> <MS>
```

说明：

- `X` 只允许 `-1 / 0 / 1`
- `Y` 只允许 `-1 / 0 / 1`
- 不能同时为 `0 0`
- `MS` 为持续时间，范围 `1` 到 `60000`

方向约定：

- `STICK 0 -1 <MS>`: 左摇杆向上
- `STICK 0 1 <MS>`: 左摇杆向下
- `STICK -1 0 <MS>`: 左摇杆向左
- `STICK 1 0 <MS>`: 左摇杆向右

示例：

```txt
STICK 0 -1 1500
STICK -1 0 3000
HOLD ZL 2500
```

含义：

- 将左摇杆推向指定方向并持续若干毫秒
- 适合直接测试自定义颜色编辑页里的模拟摇杆归位动作
- 这是原始输入命令，不带位置校准

## 7. 当前策略决策

当前建议明确固定为：

### 7.1 单色

继续作为主线开发和验证路径。

### 7.2 多色

优先使用 `基本颜色 7x12` 方案。

### 7.3 自定义颜色

先保留协议和实验能力，但不作为当前自动绘制主线。

## 8. 下一步开发建议

当前推荐的开发顺序：

1. 继续提高蓝牙连接和长时间绘制稳定性
2. 继续校准 `7 x 12` 官方色盘映射与实际观感
3. 继续优化日志、状态卡和失败恢复提示
4. 整理 Windows 试用路径与安装说明
5. 在稳定性明显提升前，不继续推进自定义颜色自动调色

## 9. 重要提醒

当前阶段不要把精力继续投入到“自定义颜色自动精确还原 hex”上，原因很明确：

- 当前开环输入不稳定
- 点按位移不线性
- 长按存在加速
- 无视觉反馈时无法保证准确复现

所以：

`基本颜色` 是当前最可落地、最工程化、最容易稳定的多色方案。
